## 1. システム概要

FC今治が運営する「里山ドッグラン」の予約・チェックイン・会員管理を一元化するWebアプリです。  
スマートフォンで操作できるモバイルファーストのUIで設計されており、会員（飼い主）と管理スタッフの両方が使えます。

### 主な機能

| 機能 | 対象 | 概要 |
|------|------|------|
| ユーザー登録・ログイン | 会員 | LINE モックによる認証（本番では LINE LIFF） |
| 犬の登録・編集・削除 | 会員 | ワクチン証明画像のアップロードとスタッフ承認 |
| 予約（日時・時間帯選択） | 会員 | カレンダーから日時選択 → オンライン決済 |
| QRコードチェックイン | 会員 | 当日現地でQRをかざして入場完了 |
| マイページ | 会員 | プロフィール・犬情報・予約履歴・決済履歴の管理 |
| 管理ダッシュボード | スタッフ | 本日の予約・売上・利用中頭数を一覧確認 |
| ワクチン証明承認 | スタッフ | 会員が提出した証明書を承認または差し戻し |
| 会員管理 | スタッフ | 連絡先・利用停止状況・no-show 回数の確認 |
| 予約管理 | スタッフ | 全予約の確認・運営側キャンセル |
| 入退場ログ | スタッフ | チェックイン記録・CSV出力 |
| 売上管理 | スタッフ | 本日売上・累計・決済待ち件数 |
| トップ表示管理 | スタッフ | トップページのスライド画像・文言を管理画面から変更 |

## 2. 画面一覧（会員向け）

### 2-1. トップページ (`/`)

- ドッグランの現在の混雑状況（空き頭数・利用中頭数）をリアルタイム表示
- 直近の自分の予約状況を表示（本日予約がある場合はチェックインへ誘導）
- トップスライダー画像（管理画面から変更可能）

### 2-2. ログイン (`/login`)

- **現在はデモ用モック認証**  
  任意の LINE User ID を入力してログインする方式（本番では LINE LIFF に変更）
- 「管理者デモアカウントでログイン」ボタンで管理画面に入れる

### 2-3. 犬登録・管理 (`/dog-registration`)

- 新しい犬の登録フォーム（名前・犬種・体重・性別・サイズ・生年月日・ワクチン情報）
- 登録済みの犬の**編集・削除**もこの画面から可能
- ワクチン証明画像をアップロードすると、スタッフの承認待ち状態になる
- **ワクチン期限切れの場合は赤いバナーで警告を表示**し、「今すぐ更新」ボタンで即時編集フォームを展開

#### ワクチン承認ステータス

| ステータス | 意味 |
|---|---|
| 確認待ち | スタッフが未確認の状態。予約不可。 |
| 承認済み | スタッフが確認済み。予約可能。 |
| 差し戻し | 証明書に問題あり。差し戻し理由を確認して再提出。 |

### 2-4. 予約 (`/reservation`)

1. **日付を選ぶ** — カレンダーから予約日を選択
2. **時間帯を選ぶ** — 空き状況をリアルタイム取得して表示
   - 満員：グレー（選択不可）
   - 残りわずか（残3頭以下）：アンバー
   - 空きあり：白
3. **犬を選ぶ** — 承認済みかつワクチン有効な犬のみチェックボックスで選択可
4. **来場人数** — 飼い主＋同伴者の人数を入力（犬の頭数以上であること）
5. **予約内容確認** — 料金（1頭200円）を表示して決済ボタン

#### 予約制約

- 利用停止中の会員は予約不可（赤バナーで通知）
- ワクチン期限切れ・未承認の犬は選択不可
- スロットの頭数上限（全体・大型犬・小型犬ごと）を超えると予約不可
- 雨天休業の場合は予約受付停止中のメッセージを表示

### 2-5. チェックイン (`/checkin`)

- カメラを起動してQRコードをかざすと自動読み取り
- 読み取れない場合はトークン手入力に対応
- チェックイン成功/失敗を画面で即時フィードバック
- カメラの停止ボタンあり

### 2-6. マイページ (`/mypage`)

- **飼い主情報**：表示名・メール・電話番号の編集
- **登録済みの犬**：ワクチン承認状況・期限切れ警告の表示。「編集」リンクで `/dog-registration` へ
- **予約履歴**：直近の予約一覧。キャンセルボタンあり（キャンセルポリシーに従い返金判定）
- **決済履歴**：過去の決済一覧（金額・状態）

### 2-7. 利用状況 (`/live-status`)

- 現在の利用頭数・空き頭数・混雑度をグラフと数値で表示（ログイン不要で閲覧可能）
- 犬種別・サイズ別の内訳表示
- 「いまいるわんこ」一覧

## 3. 管理画面（スタッフ向け）

管理画面URL: `/admin`  
**スタッフアカウント（`is_staff=true`）でのみアクセス可能。** 一般会員は閲覧・操作できない。

### 3-1. ダッシュボード (`/admin`)

- 本日の予約数・チェックイン数・利用中頭数・本日売上を表示
- 「今日の優先事項」— ワクチン承認待ち件数・決済待ち・no-show などを優先度順に一覧
- 会員数・登録犬数などのKPIカード

### 3-2. ワクチン承認 (`/admin/dogs`)

- 承認待ち / 承認済み / 差し戻し でフィルタリング
- 犬名・犬種・飼い主名で検索
- ワクチン証明画像リンクあり
- 「承認する」または「差し戻す」ボタン（審査メモを入力可能）

### 3-3. 会員管理 (`/admin/members`)

- 会員一覧（名前・メール・電話・LINE ID・登録犬数・no-show 回数・利用停止日）
- 停止中の会員は赤背景で強調
- キーワード検索対応

### 3-4. 予約管理 (`/admin/reservations`)

- 全予約の確認（日時・利用犬・決済状況・キャンセル履歴）
- 運営側でのキャンセル操作あり（理由入力必須、返金対象か否か表示）

### 3-5. 入退場ログ (`/admin/checkins`)

- チェックイン記録の一覧（日時・犬名・飼い主・利用時間・状態）
- 開始日・終了日・状態・キーワードでフィルタ
- CSV出力機能あり

### 3-6. 売上 (`/admin/sales`)

- 本日売上・決済待ち予約数・通貨別累計売上を表示

### 3-7. トップ表示管理 (`/admin/home-content`)

- トップページのスライダー画像を追加・編集・削除
- タイトル・説明文・表示順・公開/非公開の設定
- 削除前にインライン確認UI（誤操作防止）

## 5. 認証フロー

### 現在（デモ・開発用）

```
1. ユーザーが /login で LINE User ID を入力
2. POST /api/v1/auth/line  →  access_token, refresh_token, user_profile を受け取る
3. トークンを localStorage に保存
4. AuthGuard が各ページでトークンを確認
5. トークン期限切れ時は自動リフレッシュ（/auth/refresh）
```

### 本番（LINE LIFF 導入後）

```
1. LINE アプリ内で LIFF を起動
2. LINE SDK が id_token を発行
3. POST /api/v1/auth/line に id_token を送信（バックエンドで検証）
4. 以降は同じ JWT フロー
```

## 7. キャンセルポリシー

- 会員は予約をキャンセル可能（返金の可否はバックエンドのポリシーに従う）
- 無断キャンセル（no-show）が続くと予約制限あり
- 運営側でのキャンセルは管理画面 → 予約管理から実施（理由入力必須）

## 8. ワクチン管理フロー

```
会員が犬を登録
 └─ ワクチン証明画像をアップロード（または期限のみ更新）
      └─ ステータス：「確認待ち」
           └─ スタッフが /admin/dogs で画像を確認
                ├─ 承認 → ステータス：「承認済み」→ 予約可能
                └─ 差し戻し → ステータス：「差し戻し」→ 再提出が必要

ワクチン期限が切れると自動的に予約不可になり、
/dog-registration または /mypage で赤いバナーが表示される
```

## 9. 環境設定

### 環境変数

| 変数名 | 説明 | 例 |
|---|---|---|
| `NEXT_PUBLIC_API_BASE_URL` | バックエンドAPIのベースURL | `https://api.example.com/api/v1` |

`.env.local` を `frontend/` 直下に作成して設定してください。

```env
NEXT_PUBLIC_API_BASE_URL=http://localhost:8000/api/v1
```

### ローカル起動手順

```bash
# バックエンドを起動（別途 Python / Django 環境が必要）
cd backend
python manage.py runserver

# フロントエンドを起動
cd frontend
npm install
npm run dev

# ブラウザで開く
http://localhost:3000
```

### 管理者アカウントの作成

バックエンドのシードコマンドで作成するか、Django admin でユーザーの `is_staff=True` を設定してください。  
デモ環境では `/login` の「管理者デモアカウントでログイン」ボタンを使用できます。

## 10. よくある操作（スタッフ向けマニュアル）

### 新しい会員の犬を承認する

1. 管理画面 → **犬管理** (`/admin/dogs`) を開く
2. 「承認待ち」タブを選択
3. 対象の犬の「ワクチン証明画像を開く」リンクで画像を確認
4. 問題なければ「承認する」、問題があれば審査メモを入力して「差し戻す」

### 予約を運営側でキャンセルする

1. 管理画面 → **予約管理** (`/admin/reservations`) を開く
2. 対象予約の「運営でキャンセル」ボタンをクリック
3. キャンセル理由を入力して確定（返金対象かどうかが自動判定される）

### トップスライダーの画像を変更する

1. 管理画面 → **トップ表示** (`/admin/home-content`) を開く
2. 既存スライドを編集するか、下の「新規スライド追加」フォームで追加
3. 「保存」ボタンで反映（即時トップページに反映）

### 入退場記録をCSVで出力する

1. 管理画面 → **入退場ログ** (`/admin/checkins`) を開く
2. 必要に応じて開始日・終了日・状態でフィルタ
3. 右上の「CSV出力」ボタンでダウンロード
   - BOMあり UTF-8 形式（Excel でそのまま開ける）

### 11-2. 各 Azure サービスの役割

| サービス | 用途 | プラン目安 |
|---|---|---|
| **Azure Front Door** | CDN・WAF・カスタムドメイン（`dogrun.fcimabari.com` など）・HTTPS 証明書 | Standard |
| **Azure Static Web Apps** | Next.js フロントエンドのホスティング・GitHub Actions 自動デプロイ | Free / Standard |

| **Azure CDN** | Blob Storage の画像を高速配信（日本リージョン） | Standard Microsoft |
| **Azure Key Vault** | DB接続文字列・Stripe キー・LINE シークレットなど秘匿情報の一元管理 | Standard |
| **Azure Monitor + Application Insights** | APIエラー・応答時間・アクセス数の可視化・アラート | 従量課金 |

## 12. 注意事項・制限事項

1. **LINE LIFF 未実装**  
   現在はデモ用のモック認証です。本番運用には LINE LIFF の設定と、バックエンドでの id_token 検証が必要です。

2. **決済はテストモード**  
   Stripe がテスト環境の場合、実際の課金は発生しません。本番稼働前に Stripe の本番キーへの切り替えが必要です。

3. **カメラQR読み取りの対応ブラウザ**  
   BarcodeDetector API は Chrome / Android に対応。Safari（iOS）は未対応のため、手入力フォームを使用してください。

4. **画像ストレージ**  
   Azure Blob Storage を使用しています。本番環境では Azure の接続文字列設定が必要です。
